去年升学之后加入了学校的Swift Coding Club，学习了SwiftUI的基本开发，作为一名UI设计师在尝试制作一个iOS 26 iMessage风格的评论输入框时碰到了这个难绷的问题：

设置了padding，按下空格尝试拖动光标时，光标以及它的阴影会被TextField的边界切割，行数多了出现滚动之后文字也会被切割，还没有关闭的modifier。而且contentMargins只能给TextEditor使用，在TextField是无效的，我没办法在周围填充空间。

截止我写这篇博客时，我没有搜索到任何现成的解决方式，[有人提了问题](https://stackoverflow.com/questions/75742585/how-to-set-contentinset-for-swiftui-scrollable-textfield)，但是没有一个能解决问题的回答。这个问题在很多第三方聊天App中都存在，包括QQ、Signal、WhatsApp、X Chat等。但是在微信、Telegram中没有这个问题。

![X Chat中被切割的光标和文字](/blog/3/1.png)

## 解决方案

曾经我一度以为这在SwiftUI中无解，但是后来找到了有一个神奇的package叫做[SwiftUI-Introspect](https://github.com/siteline/SwiftUI-Introspect)。有一部分的SwiftUI组件其实底层是UIKit，这个package可以直接深入去控制UIKit层而不用手动重新做一个UIViewRepresentable。根据README，使用了 `axis: .vertical` 的TextField其实本质上就是一个UITextView。

在安装了这包并在需要的文件内 `import SwiftUIIntrospect` 后就可以去修改TextField底层的设置了。只需要使用以下代码就可以防止被切割：

```
.introspect(.textField(axis: .vertical), on: .iOS(.v26)) { textField in
    textField.clipsToBounds = false
    textField.textContainerInset = UIEdgeInsets(top: 9, left: 0, bottom: 9, right: 0) // 请注意无论是否需要inset都必须设置inset，你可以设置为0，不然在没有输入任何文字时按住空格移动光标可以到处乱移
    textField.showsVerticalScrollIndicator = true // 滚动时显示滚动条
}
```

需要注意的一点是需要在外部的容器上对内部内容进行 `.clipShape`，否则当文字过多需要滚动时，内容会直接超出去，以下为完全还原iMessage的modifier，先后顺序不能反，不然 `.glassEffect` 会被切割：

```
.clipShape(RoundedRectangle(cornerRadius: 20))
.glassEffect(.regular.interactive(), in: .rect(cornerRadius: 20))
```
